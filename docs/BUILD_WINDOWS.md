# Windows desktop build (clean git clone)

Contributor guide for building `SSC-Setup-*.exe` on **Windows 10/11** from a fresh clone. No maintainer DMs or production secrets required.

## Prerequisites

| Tool | Version | Notes |
|------|---------|--------|
| **Git** | any recent | `git clone https://github.com/raullavita/SSC.git` |
| **Node.js** | **22.x** (CI uses 22) | `node -v` |
| **Yarn** | 1.x | `corepack enable` or `npm i -g yarn` |
| **Java JDK** | 17 or 21 | Only needed for **Android** builds, not desktop |

Desktop build does **not** need Docker, MongoDB, or a filled `backend/.env`.

## One-command build

From the repo root:

```powershell
.\SSC-BUILD-DESKTOP-WIN.bat
```

This calls the canonical script: `scripts/build/build_desktop_win.bat`.

### What the script does

1. `yarn build:desktop` — production React bundle for Electron
2. `yarn install` in `frontend/desktop/`
3. `yarn build:win` — packages `SSC-Setup-1.0.12.exe` (version from `frontend/package.json`)

Signing is **off** unless you set `CSC_LINK` (code-signing cert). Unsigned builds are fine for local QA; Windows SmartScreen may warn once.

## Output locations

| Path | Purpose |
|------|---------|
| `frontend/desktop/dist/SSC-Setup-1.0.12.exe` | Build artifact inside the repo |
| `C:\Users\smash\Desktop\SSC\SSC-Setup-1.0.12.exe` | Convenience copy (maintainer path in script) |

Contributors can use `frontend/desktop/dist/` only — edit the `copy` line in `scripts/build/build_desktop_win.bat` if you prefer another folder.

## Gitignored files you do **not** need

These are **not** in the public repo and are **not** required for a contributor desktop build:

| File / folder | Why gitignored |
|---------------|----------------|
| `frontend/node_modules/` | Restored by `yarn install` (run automatically in build) |
| `frontend/desktop/node_modules/` | Restored by desktop `yarn install` |
| `frontend/build/` | Generated by `yarn build:desktop` |
| `frontend/desktop/dist/` | Installer output |
| `backend/.env`, `frontend/.env` | Runtime secrets — desktop packages use baked production API URL |
| `frontend/android/keystore.properties`, `*.jks` | Android signing only |
| `frontend/public/downloads/*.apk`, `*.exe` | Hosting deploy copies — see `scripts/prepare_downloads.ps1` |

## Troubleshooting

- **`yarn` not found** — install Node 22, then `corepack enable` and `corepack prepare yarn@stable --activate`.
- **Build fails on `build:desktop`** — run `cd frontend && yarn install` once, then retry the bat file.
- **SmartScreen blocks install** — expected without a code-signing cert; click “More info” → “Run anyway” for local testing.

SSC + Firebase (Spark plan) — connection guide
==============================================
Project: super-chat-b0992  |  Android package: chat.ssc.secure

IMPORTANT — what Firebase IS and IS NOT
-----------------------------------------
Firebase Spark (free) gives you:
  - FCM push (Android)          — ALREADY wired in SSC
  - App Distribution (testers)  — upload APK, invite by email
  - Hosting (optional PWA web)  — static site from frontend/build
  - Google Cloud project shell  — same account for Cloud Run API

Firebase does NOT run the Python/FastAPI backend.
The API must be HTTPS somewhere else (Cloud Run, Render, Railway).
MongoDB stays on Atlas (already in backend/.env).

STEP B — Set production API URL in frontend
-------------------------------------------
Copy frontend/.env.production.local.example → .env.production.local

  REACT_APP_BACKEND_URL=https://YOUR-DEPLOYED-API-URL

No trailing slash. Must be https:// for App Tester / Play-bound builds.

STEP C — Backend CORS (backend/.env on deployed server)
-------------------------------------------------------
Add your origins (comma-separated, no spaces issue):

  CORS_ORIGINS=https://YOUR-API-URL,https://localhost,capacitor://localhost,https://super-chat-b0992.web.app

https://localhost is required for Capacitor Android (androidScheme: https).
capacitor://localhost is kept for legacy shells.

STEP D — Build APK for Firebase App Distribution
------------------------------------------------
  cd frontend
  yarn build:firebase
  yarn cap sync
  cd android
  gradlew.bat assembleDebug

APK path:
  frontend/android/app/build/outputs/apk/debug/app-debug.apk

Upload in Firebase Console → App Distribution → Releases.

STEP E — Install Firebase CLI (optional, for hosting + CI uploads)
------------------------------------------------------------------
  npm install -g firebase-tools
  firebase login
  firebase use super-chat-b0992

Host PWA:
  cd frontend && yarn build:firebase
  cd .. && firebase deploy --only hosting
